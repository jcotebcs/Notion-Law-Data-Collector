Processing Status

After submitting case data, you can monitor the processing status by:

  • Checking the GitHub issue that gets created
  • Viewing the Actions tab in your repository
  • Looking for automatic comments on the issue with results

The GitHub Actions workflow will automatically close the issue once the data is successfully added to Notion.

Setup Instructions

1. Setup Repository Secrets

  1. Go to your repository Settings > Secrets and variables > Actions
  2. Add NOTION_TOKEN with your Notion integration token
  3. Add NOTION_DATABASE_ID with your database ID
  4. Get your Notion API key from Notion Integrations

2. Create Issue Label

  1. Go to your repository Issues > Labels
  2. Create a new label called legal-entry
  3. This label is required for the GitHub Actions workflow to process submissions

3. Create a Database

  1. Create a new page in Notion
  2. Add a database with these properties:
    • Title (Title)
    • Case Number (Rich Text)
    • Court (Rich Text)
    • Judge (Rich Text)
    • Date (Date)
    • Status (Select)
    • Parties (Rich Text)
    • Type (Select)
    • Summary (Rich Text)
    • Outcome (Rich Text)
    • Tags (Multi-select)
    • Priority (Select)
  3. Share the database with your integration
  4. Copy the database ID from the URL

4. Configure This App

  1. Enter your GitHub repository information above
  2. Paste your database ID
  3. Test the configuration
  4. Start adding case data!
